# Upgrading from v0.1.0 to v0.2.0

v0.2.0 is a **breaking change**. The chart structure changes significantly — OpenWebUI and LiteLLM
are now deployed via official upstream Helm subcharts rather than InferenceHub's own templates.
Additionally, the single `inferencehub` PostgreSQL database is split into two: `openwebui` and
`litellm`.

You cannot `helm upgrade` from v0.1.0 without first running the database migration.

---

## What changed

| Area | v0.1.0 | v0.2.0 |
|------|--------|--------|
| OpenWebUI deployment | InferenceHub template | open-webui subchart (v12.5.0) |
| LiteLLM deployment | InferenceHub template | litellm-helm subchart (v1.1.0) |
| PostgreSQL databases | single `inferencehub` DB | separate `openwebui` and `litellm` DBs |
| LiteLLM + PostgreSQL | LiteLLM not connected to DB | LiteLLM connected by default |
| Config passthrough | limited via fixed values | full passthrough via `openwebui:` and `litellm:` keys |

---

## Upgrade steps

### Step 1: Back up the database

```bash
kubectl exec -n inferencehub \
  $(kubectl get pod -n inferencehub -l app.kubernetes.io/component=database -o jsonpath='{.items[0].metadata.name}') \
  -- pg_dump -U inferencehub inferencehub > inferencehub-backup-$(date +%Y%m%d).sql
```

Verify the backup is non-empty before proceeding.

### Step 2: Run the migration script

The migration script creates the `openwebui` and `litellm` databases and copies OpenWebUI's
tables into `openwebui`. LiteLLM tables are left empty (LiteLLM recreates them on startup from
an empty database).

```bash
# Port-forward PostgreSQL so the script can reach it
kubectl port-forward -n inferencehub \
  $(kubectl get pod -n inferencehub -l app.kubernetes.io/component=database -o jsonpath='{.items[0].metadata.name}') \
  5432:5432 &
PF_PID=$!

# Run the migration (requires psql on your local machine)
PGPASSWORD=<your-postgres-password> bash docs/upgrading/migrate-database.sh

kill $PF_PID
```

### Step 3: Update your `inferencehub.yaml`

If you previously set values under the `postgresql.external` key, update the format:

```yaml
# v0.1.0 (old)
postgresql:
  external:
    connectionString: "postgresql://user:pass@host:5432/inferencehub"

# v0.2.0 (new)
postgresql:
  external:
    enabled: true
    openwebuiConnectionString: "postgresql://user:pass@host:5432/openwebui"
    litellmConnectionString:   "postgresql://user:pass@host:5432/litellm"
```

### Step 4: Run the upgrade

```bash
inferencehub upgrade --config inferencehub.yaml
```

### Step 5: Verify

```bash
inferencehub verify
kubectl get pods -n inferencehub
```

Both `openwebui` and `litellm` pods should reach `Running` state within a few minutes.

## Rollback

If you need to roll back:

```bash
helm rollback inferencehub 1 -n inferencehub
```

Then restore the database backup:

```bash
kubectl exec -i -n inferencehub \
  $(kubectl get pod -n inferencehub -l app.kubernetes.io/component=database -o jsonpath='{.items[0].metadata.name}') \
  -- psql -U inferencehub < inferencehub-backup-<date>.sql
```
